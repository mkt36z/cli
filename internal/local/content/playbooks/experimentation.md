## The Core Principle

> Speed of testing > quality of guessing. The best marketers don't have better ideas — they test more ideas, faster, and learn from the results.

This framework ensures every marketing experiment has a clear hypothesis, measurable outcome, and documented learning — so you compound knowledge over time instead of repeating the same mistakes.

---

## Part 1: The Experiment Process

### The 5-Step Experiment Loop

```
1. OBSERVE → What data/feedback suggests an opportunity?
2. HYPOTHESIZE → What change do we believe will improve results?
3. DESIGN → How do we test this with minimum effort?
4. EXECUTE → Run the test with proper measurement
5. ANALYZE → What happened? What did we learn?
    ↓
   Loop back to Step 1 with new insights
```

## Part 3: Experiment Prioritization

### The ICE Scoring Framework

Score each potential experiment on 3 dimensions (1-10 scale):

| Dimension | Question | Score |
|-----------|----------|:-----:|
| **Impact** | If this works, how big is the effect on our key metric? | /10 |
| **Confidence** | How confident are we that this will work? (based on data, not hope) | /10 |
| **Ease** | How easy is this to implement and measure? | /10 |

**ICE Score = (Impact + Confidence + Ease) / 3**

### Experiment Backlog (Prioritized)

| # | Experiment Name | Category | Impact | Confidence | Ease | ICE Score | Status |
|---|----------------|----------|:------:|:----------:|:----:|:---------:|:------:|
| 1 | | | /10 | /10 | /10 | | Backlog / Active / Complete |
| 2 | | | | | | | |
| 3 | | | | | | | |
| 4 | | | | | | | |
| 5 | | | | | | | |

### Prioritization Rules

1. **Run max 2-3 experiments simultaneously** (solo founder can't manage more)
2. **High-impact experiments first** (even if lower confidence)
3. **Quick wins for momentum** (mix in some high-ease experiments)
4. **Never run conflicting experiments** (two tests on the same page/audience)
5. **Revenue experiments before vanity experiments** (conversion rate > follower count)

## Part 4: Common Marketing Experiments

### Acquisition Experiments

| Experiment | What to Test | Metric | Typical Impact |
|-----------|-------------|--------|:--------------:|
| **Headline A/B test** | 2-3 headline variants on landing page | Conversion rate | 10-50% |
| **Hook test (ads)** | 5-10 hook variants, same body/CTA | CTR, CPC | 20-100% |
| **Channel test** | New distribution channel (e.g., Reddit, newsletter sponsorship) | CAC, lead volume | Variable |
| **Lead magnet test** | Different lead magnet topics or formats | Signup rate | 15-40% |
| **CTA test** | Button text, color, placement | Click-through rate | 5-30% |
| **Social post format** | Text vs. carousel vs. video vs. poll | Engagement, reach | 20-80% |

### Activation Experiments

| Experiment | What to Test | Metric | Typical Impact |
|-----------|-------------|--------|:--------------:|
| **Onboarding email sequence** | Different welcome sequence (length, content, CTA) | Activation rate | 10-30% |
| **First-use experience** | Simplify first action, add tutorial, change default | Time to value | 15-40% |
| **Pricing page layout** | Tier presentation, anchoring, feature display | Plan selection, conversion | 10-25% |

### Revenue Experiments

| Experiment | What to Test | Metric | Typical Impact |
|-----------|-------------|--------|:--------------:|
| **Price point test** | Higher vs. lower price on same offer | Revenue per visitor | 10-50% |
| **Upsell timing** | When to present upsell (checkout, email, in-app) | Upsell conversion | 5-20% |
| **Guarantee test** | With guarantee vs. without, or different guarantee types | Conversion rate, refund rate | 10-30% |
| **Value stack test** | Different bonus combinations | Perceived value, conversion | 10-25% |

### Retention Experiments

| Experiment | What to Test | Metric | Typical Impact |
|-----------|-------------|--------|:--------------:|
| **Re-engagement email** | Different re-engagement subject lines and offers | Reactivation rate | 5-15% |
| **Usage nudge** | Email/notification when usage drops | Weekly active rate | 10-20% |
| **Feature education** | Highlight underused features | Feature adoption, retention | 5-15% |

## Part 5: Running the Experiment

### Pre-Launch Checklist

- [ ] Hypothesis written (specific, measurable)
- [ ] Success metric defined
- [ ] Minimum sample size calculated
- [ ] Minimum duration set (usually 1-2 weeks minimum)
- [ ] Tracking/analytics in place
- [ ] Control and variant clearly defined
- [ ] No other experiments running on same audience/page
- [ ] Kill criteria defined

### During the Experiment

| Action | Frequency |
|--------|:---------:|
| Check data for anomalies (tracking broken, etc.) | Daily (first 3 days) |
| Monitor for external factors (holidays, PR, outages) | Daily |
| Resist the urge to call it early | Ongoing discipline |
| Document observations | As they occur |

### Common Mistakes

| Mistake | Why It's Bad | What to Do Instead |
|---------|-------------|-------------------|
| Ending the test too early | Results may not be statistically significant | Wait for minimum sample size |
| Testing too many variables | Can't attribute results to a single change | Change ONE thing per test |
| No control group | No baseline to compare against | Always have a control |
| Ignoring external factors | A sale or holiday skews results | Note external events, extend test if needed |
| Not documenting learnings | Same mistakes repeated | Complete the experiment review every time |

## Part 8: Building an Experimentation Culture

### The Experiment Cadence (Solo Founder)

| Timeframe | Target |
|-----------|:------:|
| Weekly | Run 1-2 small experiments (email subject lines, social hooks) |
| Biweekly | Run 1 medium experiment (landing page, ad creative) |
| Monthly | Run 1 large experiment (pricing, offer, new channel) |
| Quarterly | Review all learnings, update strategy based on data |

### The Experimentation Mindset

1. **Every marketing decision is a hypothesis** — treat it as such
2. **Failure is data** — a "losing" experiment is valuable if you learn from it
3. **Small bets, big learnings** — test cheaply before scaling
4. **Document everything** — your experiment log is your competitive advantage
5. **Compound learnings** — each experiment informs the next one

## Part 9: A/B Test Statistical Rigor

*Source: MarketingSkills `ab-test-setup` skill*

### Hypothesis Structure

```
Because [observation/data],
we believe [specific change]
will cause [specific metric] to [increase/decrease] by [estimated amount]
for [target audience].
We'll know when [primary metric reaches X with Y confidence].
```

### Sample Size Quick Reference

| Baseline Conversion Rate | Minimum Detectable Effect (MDE) | Sample Size Per Variant |
|:------------------------:|:-------------------------------:|:----------------------:|
| 1% | 50% lift (→1.5%) | ~23,000 |
| 2% | 25% lift (→2.5%) | ~14,000 |
| 5% | 20% lift (→6%) | ~7,000 |
| 10% | 15% lift (→11.5%) | ~5,000 |
| 20% | 10% lift (→22%) | ~4,000 |
| 50% | 5% lift (→52.5%) | ~6,000 |

*At 95% confidence, 80% power. Use an online calculator for exact numbers.*

### Metric Selection

| Metric Type | Purpose | Example |
|------------|---------|---------|
| **Primary** (exactly 1) | Tied directly to hypothesis — determines success/failure | Conversion rate |
| **Secondary** (1-3) | Provides context, helps explain primary | Revenue per visitor, bounce rate |
| **Guardrail** (1-2) | Should NOT get worse — protects against unintended harm | Page load time, support tickets |

### Traffic Allocation

| Strategy | Split | When to Use |
|----------|:-----:|-------------|
| Standard | 50/50 | Default for most tests |
| Conservative | 90/10 or 80/20 | High-risk changes, revenue-critical pages |
| Ramping | 10% → 25% → 50% | New features with unknown impact |

### The Peeking Problem

**DO NOT stop tests early based on interim results.** Peeking at results and stopping when you see a "winner" causes inflated false positive rates (up to 30%+ instead of 5%).

**Rules:**
- Pre-commit to sample size before starting
- Set a minimum test duration (at least 1-2 full business cycles)
- Only read results after reaching the committed sample size
- If you must peek, use sequential testing methods (but understand the tradeoffs)

### Implementation Types

| Type | How It Works | Pros | Cons |
|------|-------------|------|------|
| **Client-side** | JavaScript swaps elements in browser | Easy to implement, no dev needed | May cause flicker, slower |
| **Server-side** | Server renders the correct variant | No flicker, more reliable | Requires dev work |

## Part 10: Analytics Tracking Event Taxonomy

*Source: MarketingSkills `analytics-tracking` skill*

### The Core Principle

> **Track for decisions, not data.** Start with the questions you need to answer, then work backwards to the events that answer them.

### Event Naming Convention

**Format:** `object_action` — lowercase, underscores, past tense for completed actions

| Category | Event Name | When to Fire |
|----------|-----------|-------------|
| **Marketing Site** | `cta_clicked` | Any CTA button clicked (with `location` property) |
| | `form_submitted` | Contact/demo/lead form submitted |
| | `signup_started` | Signup flow initiated |
| | `signup_completed` | Account successfully created |
| **Product** | `onboarding_step_completed` | Each onboarding step (with `step_number` and `step_name`) |
| | `feature_used` | Core feature activated (with `feature_name`) |
| | `campaign_created` | New campaign started |
| | `campaign_completed` | Campaign fully executed |
| | `upgrade_clicked` | Upgrade CTA clicked |
| | `purchase_completed` | Subscription purchased or upgraded |

### UTM Parameter Strategy

| Parameter | Purpose | Convention | Example |
|-----------|---------|-----------|---------|
| `utm_source` | Where traffic comes from | Platform name, lowercase | `linkedin`, `google`, `newsletter` |
| `utm_medium` | Marketing medium | Channel type | `cpc`, `organic`, `email`, `social` |
| `utm_campaign` | Campaign identifier | Descriptive, hyphens | `spring-launch-2026` |
| `utm_content` | Creative variant | Variant identifier | `headline-a`, `cta-green` |
| `utm_term` | Keyword (paid search) | Keyword or audience | `ai-marketing-tool` |

**Rules:**
- Always lowercase
- Document all campaigns in a shared spreadsheet
- Never change conventions mid-campaign

### GA4 + GTM Implementation Pattern

```javascript
// Push events to the data layer for GTM to capture
dataLayer.push({
  event: 'signup_completed',
  method: 'email',           // or 'google', 'github'
  plan: 'growth',
  source: 'pricing_page'
});
```

### Privacy & Compliance

- Cookie consent required for EU/UK/CA visitors
- No PII (personally identifiable information) in event properties
- IP anonymization enabled
- Consent mode for GA4 (restricted data collection before consent)
- Document data retention policies
