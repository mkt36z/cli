## Part 1: Crisis Classification

### The Crisis Severity Matrix

| Level | Severity | Examples | Response Time | Escalation |
|:-----:|:--------:|---------|:-------------:|-----------|
| **1** | Minor | Negative review, small social media complaint | 24 hours | Marketing team |
| **2** | Moderate | Product bug affecting users, minor PR mention | 4 hours | Marketing + Product |
| **3** | Serious | Data breach, major product failure, viral negative press | 1 hour | C-suite + Legal + Marketing |
| **4** | Critical | Legal action, regulatory investigation, safety issue | Immediate | CEO + Legal + Board + PR |

### Crisis Type Identification

| Crisis Type | Description | Example | Default Response Posture |
|------------|-----------|---------|------------------------|
| **Product/Service failure** | Your product/service causes harm or fails | Outage, data loss, security breach | Transparent accountability |
| **Reputation attack** | External attack on your brand or leadership | Hit piece, viral complaint, competitor smear | Factual defense + values reinforcement |
| **Employee misconduct** | Internal behavior becomes public | Harassment, fraud, inappropriate content | Swift action + organizational values |
| **Industry crisis** | Market-wide event affects your category | Regulation change, industry scandal | Differentiation + reassurance |
| **Customer crisis** | Customer misuses your product/service publicly | Misrepresentation, public failure | Empathy + clarification of responsibility |

## Part 2: Crisis Response Framework

### The CARE Framework

| Step | Action | Timing |
|------|--------|:------:|
| **C — Contain** | Stop the bleeding. Prevent the situation from escalating. | Immediate |
| **A — Acknowledge** | Publicly acknowledge the issue. Silence is interpreted as guilt. | Within 1-4 hours |
| **R — Resolve** | Fix the problem and communicate the fix. | As fast as possible |
| **E — Evolve** | Document learnings and implement changes to prevent recurrence. | Within 1-2 weeks |

### Phase 1: First 60 Minutes (Contain)

| Action | Owner | Status |
|--------|-------|:------:|
| Assemble crisis team (see roles below) | CEO / CMO | |
| Assess severity level (1-4) | Crisis Lead | |
| Gather facts — what happened, who's affected, what do we know | All | |
| Pause all scheduled marketing (social posts, emails, ads) | Marketing | |
| Draft holding statement | Communications | |
| Brief customer support with approved responses | Customer Support | |
| Notify legal counsel | CEO / COO | |

### Phase 2: Hours 1-4 (Acknowledge)

| Action | Owner | Status |
|--------|-------|:------:|
| Publish holding statement on primary channels | Communications | |
| Notify affected customers directly (if identifiable) | Customer Support | |
| Monitor social media and press mentions | Marketing | |
| Prepare detailed response / FAQ | Communications + Legal | |
| Brief sales team on talking points | Sales Leadership | |
| Set up response monitoring cadence (hourly → daily) | Marketing | |

### Phase 3: Hours 4-48 (Resolve)

| Action | Owner | Status |
|--------|-------|:------:|
| Publish detailed response with actions taken | Communications | |
| Implement the fix (product, process, policy) | Responsible team | |
| Respond to press inquiries with approved statements | PR / Communications | |
| Direct customer outreach for most affected parties | CEO + Customer Success | |
| Update stakeholders (board, investors, partners) | CEO | |
| Monitor sentiment shift | Marketing | |

### Phase 4: Days 2-14 (Evolve)

| Action | Owner | Status |
|--------|-------|:------:|
| Conduct internal postmortem | Crisis Lead | |
| Publish "what we learned" / transparency update | Communications | |
| Implement preventive measures | Responsible team | |
| Resume normal marketing operations (gradually) | Marketing | |
| Follow up with affected customers | Customer Success | |
| Update crisis playbook based on learnings | Communications | |

## Part 3: Crisis Communication Templates

### Holding Statement (First Response)

```
We are aware of [brief description of the issue] and are actively investigating.

[Number of affected users/scope, if known.]

Our team is working urgently to [immediate action being taken].

We take this seriously and will provide a detailed update within [timeframe].

If you've been affected, please [specific action — contact support, etc.].

We'll continue to update this page as we learn more.
```

### Detailed Response (Follow-Up)

```
UPDATE [Date, Time]:

On [date], we [what happened — factual, specific, no spin].

What we know:
• [Fact 1]
• [Fact 2]
• [Fact 3]

What we've done:
• [Action 1 — immediate fix]
• [Action 2 — customer remediation]
• [Action 3 — preventive measure]

What happens next:
• [Next step + timeline]
• [How we'll communicate updates]

We understand this [affected your trust / caused disruption / fell short of
expectations]. [Genuine accountability statement — no deflection.]

If you have questions: [specific contact / channel].

— [Name, Title]
```

### Social Media Response (Short-Form)

```
We're aware of [issue]. Our team is investigating and we'll share an update
within [timeframe]. Details: [link to status page or full statement].
```

## Part 4: Crisis Team Roles

| Role | Responsibility | Who |
|------|---------------|-----|
| **Crisis Lead** | Decision-making authority, coordinates response | CEO or designated executive |
| **Communications Lead** | Drafts all external messaging, manages media | Head of Comms / CMO |
| **Legal Advisor** | Reviews all statements, advises on liability | General Counsel |
| **Technical Lead** | Investigates root cause, implements fix | CTO / Head of Engineering |
| **Customer Support Lead** | Manages frontline customer communication | Head of Support |
| **Social Media Monitor** | Tracks mentions, sentiment, escalating conversations | Social Media Manager |
| **Internal Comms** | Keeps employees informed and aligned | HR / CEO |

## Part 5: Communication Do's and Don'ts

| Do | Don't |
|----|-------|
| Acknowledge quickly — even if you don't have all the facts | Don't go silent — silence is interpreted as indifference or guilt |
| Be specific about what you know and don't know | Don't speculate or over-promise |
| Show genuine empathy | Don't use corporate jargon ("we take this seriously" with no action) |
| Take responsibility where appropriate | Don't blame customers, partners, or employees publicly |
| Provide a clear next step for affected people | Don't make people hunt for information |
| Communicate through the crisis (updates) | Don't make one statement and disappear |
| Let leadership be the face of the response | Don't hide behind a faceless corporate statement |
| Be honest if you made a mistake | Don't try to minimize or deflect |

## Part 6: Pre-Crisis Preparation

### The Crisis Preparedness Checklist

- [ ] Crisis team identified with contact info (24/7)
- [ ] Communication channel for crisis team (Slack channel, Signal group)
- [ ] Pre-drafted holding statement templates (customize per scenario)
- [ ] Social media monitoring tool configured with brand alerts
- [ ] Press/media contact list (journalists who cover your space)
- [ ] Legal counsel on retainer or quick-access arrangement
- [ ] Customer communication channels identified (email, in-app, status page)
- [ ] Designated spokesperson trained for media interactions
- [ ] Crisis simulation run at least once per year
- [ ] Status page or equivalent set up (even if never used)

## Part 7: Post-Crisis Recovery

### Rebuilding Trust Timeline

| Timeframe | Action |
|-----------|--------|
| Week 1 | Publish "what we learned" transparency post |
| Week 2-4 | Direct outreach to affected customers with remediation |
| Month 2 | Share progress on preventive measures |
| Month 3 | Publish case study of how you improved (if appropriate) |
| Ongoing | Over-communicate product reliability, security, and quality |
