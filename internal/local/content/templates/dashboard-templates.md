## Dashboard Setup Checklist

### Before Building Any Dashboard

- [ ] Define the audience (who will read this?)
- [ ] Define the cadence (how often will it be reviewed?)
- [ ] Define the decisions it should inform (what actions will data drive?)
- [ ] Identify data sources and verify access
- [ ] Agree on metric definitions (e.g., what counts as a "lead"?)
- [ ] Set targets/benchmarks for every metric
- [ ] Establish who owns the update process

### Dashboard Design Principles

1. **One page per audience** — if they have to scroll, you've lost them
2. **Lead with the story, not the data** — start with "what happened and why"
3. **Red/Yellow/Green everything** — executives scan, they don't analyze
4. **Show trends, not snapshots** — a single number is meaningless without context
5. **Always include "so what?"** — every metric should have an implied next action
